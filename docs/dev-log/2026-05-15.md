## Key lessons / patterns (so we don't relearn them)

### react-pdf has no CSS gradients

Gradients must go through react-pdf's SVG primitives: `Svg` + `Defs` +
`LinearGradient`/`RadialGradient` + `Stop`, filling a `Rect`/`Circle`. `viewBox="0 0 100 100"`
+ `preserveAspectRatio="none"` + absolute-positioned `Svg` works for decorative orbs/bands.
(All removed in the end, but the technique is in git history at `9a9a16b`/`f3f928b`.)

### Verifying react-pdf without a running app

`server-only` is a Next bundler virtual module — a standalone `tsx` script can't import the
PDF component. Vitest already aliases `server-only` to a stub (`vitest.config.ts`), so the
canonical verification is a vitest test that calls `renderToBuffer`, writes to `/tmp`, then
`pdftoppm -r 120 -png` + visually inspect. Build passing also proves it compiles under Next.

### Stitch MCP is a mockup tool, not a code source

Stitch outputs web/mobile screen designs (HTML/CSS). It cannot produce react-pdf. Use it for
visual direction only, then hand-translate into the restricted react-pdf style subset. A
print-targeted design system + explicit "this is an A4 print page, no buttons/nav" prompt
gave usable references.

### Stretched-link pattern for clickable rows with controls

To keep a whole list row clickable while embedding an interactive control: `<li relative>` +
`<Link absolute inset-0 aria-label=…>` (no children) + content wrapper `relative
pointer-events-none` + the interactive cell `pointer-events-auto relative z-10`. Clicks on
empty area fall through to the link; the control captures its own clicks. Standard, accessible.

### jsdom component tests

Harness now exists. Add `// @vitest-environment jsdom` as the first line, mock
`next/navigation` + the relevant `@/server/actions/*` modules, render with
`@testing-library/react`. Used by `customer-form-nav` and `status-select`. `tests/setup.ts`
already pulls in jest-dom matchers.

### Recurring false positive

The Next.js post-edit validator hook keeps recommending `await params` on the quotes/invoices
list pages. Those pages only use `searchParams` (already awaited) and have no route `params`.
Ignore it for `src/app/admin/(authed)/{quotes,invoices}/page.tsx`.
